## 🎯 Core Vision

**AutoTestX** is designed to reshape the landscape of the software testing industry. We are not just building a tool; we are driving a revolution in efficiency and professionalization:

> [!IMPORTANT]
> **Empowering Manual Testers**: Enabling entry-level testers who currently rely on manual repetitive clicking to instantly possess the capability to build complex automated tests through natural language. Tools are no longer the barrier; business logic is the core.

> [!WARNING]
> **Eliminating the "Middle Layer"**: We aim to replace the "middle layer" of testers who can write basic scripts but lack true software engineering skills. AutoTestX's AI capabilities will fully cover this inefficient script-maintenance work.

> [!TIP]
> **Advocating for True Test Development**: Forcing quality professionals to break free from low-quality script writing and transition into true **Software Development Engineers in Test (SDET)**. Spend your time researching underlying test infrastructures and dev-tooling—returning to the essence of engineering.

## 🚀 Core Features

| Feature | Description |
| :--- | :--- |
| **Visual Case Management** | Waterfall management of test suites with full CRUD support and assertion definitions. |
| **Natural Language Driven** | Powered by [qa-use](https://github.com/browser-use/qa-use), watch the AI Agent control the browser in real-time. |
| **Intelligent Scheduler** | Built-in Cron engine supporting standard expressions for 7x24 automated regression runs. |
| **Deep Test Insights** | Every action, history trace, and duration recorded for comprehensive auditing. |

## 🛠️ Getting Started

### 1. Initialize Environment
```bash
git clone https://github.com/YourUsername/AutoTestX.git
cd AutoTestX
pnpm run init
```
> *The `init` script automatically configures Node dependencies and the Python `uv` runtime.*

### 2. Run Development Mode
```bash
pnpm dev
```

### 3. Configure Models
After launching, navigate to the **"Model Settings"** module to input your OpenAI or Claude API keys.
